# hh — автоматизация поиска работы на hh.ru

Агентная автоматизация полного цикла соискателя на [hh.ru](https://hh.ru), управляемая из [Claude Code](https://docs.anthropic.com/en/docs/claude-code): скан вакансий → LLM-скоринг под профиль → сопроводительные письма → отклики (включая вакансии с обязательными тестами) → разбор входящих от работодателей → авто-ответы на опросники ботов и тех-скрининги → сводные отчёты.

Python здесь — только механика (HTTP, парсинг, фильтры по спискам). Все решения — скоринг, письма, ответы на вопросы, классификация сообщений — принимает LLM внутри скиллов Claude Code.

## Как использовать

Открой Claude Code в корне проекта (Chrome залогинен на hh.ru) и дай один промпт на весь дневной цикл. Первый промпт реальной сессии, дословно:

> ответь на новые вакансии появившиеся на hh.ru, пройди опросники и тесты если потребуется. Позови другого фейл сабагента — чтобы ответил на тесты и технические вопросы в личных сообщениях от рекрутеров. Составь список тех компаний, анализируя личные сообщения, которые мне предлагают пройти интервью, скрининг или созвониться/списаться в мессенджере — составь таблицу с такими компаниями/вакансиями. На личные ответы, которые писали люди (не ИИ-помощники), не отвечай сам — и подготовь мне список по ним тоже

Что из него получилось за один прогон (2026-07-15):
- скан 270 вакансий → 45 новых после дедупа → LLM-скоринг в 5 параллельных сабагентов;
- 12 откликов с персональными письмами (10 POST + 2 с прохождением тестов в headed-браузере), все верифицированы по negotiations;
- входящие: авто-прохождение бот-анкеты, ответ на тех-скрининг из 6 вопросов, классификация отказов;
- итоговые таблицы: кто зовёт на интервью, что ждёт решения человека.

Дальше цикл поддерживается короткими промптами: «проверь ответы работодателей» (`/hh-inbox`), «проскань hh» (`/hh-scan`), «пройди тесты» (`/hh-test-apply`).

## Как это работает

```
┌──────────────────────────────────────────────────────────┐
│  /hh-scan  (skill — главная точка входа)                  │
│  scan → enrich → score(LLM) → report → CONFIRM → apply    │
└───────────────┬──────────────────────────────────────────┘
                │            ┌──────────────────────────────┐
                │            │  /hh-test-apply — вакансии   │
                │            │  с тестом: headed Playwright │
                │            ├──────────────────────────────┤
                │            │  /hh-inbox — входящие:       │
                │            │  ИИ-опросники → авто-ответ;  │
                │            │  люди → согласование         │
                │            └──────────────────────────────┘
   ┌────────────▼─────────────┐
   │ src/hh_session.py        │  куки Chrome (browser_cookie3) + парсер HH-Lux-InitialState
   │ src/scan.py              │  поиск по запросам/регионам → префильтр → vacancies.json
   │ src/fetch_vacancy.py     │  полные тексты вакансий → enriched.json
   │ src/apply.py             │  POST-отклики с письмами, dry-run по умолчанию
   │ src/inbox.py             │  входящие: все отклики (negotiations) + REST-история чатов
   │ src/chat_send.py         │  ответ в чат REST'ом: dry-run, race-check, статус-гейты
   │ src/chat_leave.py        │  чистка чатов (только по команде человека)
   │ src/applied_log.py       │  журнал переписки: enum статусов, flock, атомарная запись
   └──────────────────────────┘
```

Авторизация — куки залогиненного Chrome через `browser_cookie3`: отдельных токенов и OAuth не нужно. Текстовые ответы в чаты уходят REST'ом (обязательный dry-run, race-check на новые сообщения, гейты по статусу переписки); прохождение тестов и анкет — headed Playwright под управлением сабагента (opus/fable). Любая команда с `--send` дополнительно требует подтверждения человека через permissions-хук. Ответы — строго по фактам из `profile/candidate-facts.md`.

## Точечные команды

```bash
python3 src/scan.py                                      # скан → data/vacancies.json
python3 src/fetch_vacancy.py --limit 40                  # полные тексты → data/enriched.json
python3 src/apply.py --input data/to-apply.json          # dry-run откликов
python3 src/apply.py --input data/to-apply.json --send   # реальная отправка
python3 src/inbox.py                                     # входящие: сводка + data/inbox.json
python3 src/chat_send.py --chat-id N --text-file f --expect-last-id M [--send]  # ответ в чат
```

## Структура

- `.claude/skills/` — пайплайны (`hh-scan`, `hh-test-apply`, `hh-inbox`) с выстраданными деталями протокола hh.ru
- `profile/profile.md` — критерии скоринга: стек, штрафы, стоп-лист компаний, шкала 0–10
- `profile/candidate-facts.md` — источник истины для ответов на анкеты и вопросы рекрутёров
- `data/` — состояние: дедуп, скоринг, лог откликов со статусами и диалогами, отчёты по дням
- `CLAUDE.md` — архитектура, границы python/LLM, протокольные ловушки hh.ru (406-на-отклик, rolling-лимиты, captcha-стены)

## Принципы

- **Отклики уходят только после подтверждения человека**; dry-run всегда первым.
- **Честность ответов**: факта нет в `candidate-facts.md` → честное «нет» или вопрос возвращается человеку; выдуманный опыт хуже отсутствия ответа.
- **Троттлинг** всех запросов к hh.ru; скрипты останавливаются по маркерам captcha/лимитов.
- Человеческие сообщения от рекрутёров не получают автоматических ответов — только согласованные.

## Требования

Python 3.12+, `requests`, `browser_cookie3`, `playwright` (+ Chrome), залогиненный на hh.ru Chrome-профиль, Claude Code со скиллами из `.claude/skills/`.
