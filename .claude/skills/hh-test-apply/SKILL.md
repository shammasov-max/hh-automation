# hh-test-apply: отклики с тестами (headed Playwright + opus)

Рабочий каталог: `/home/max/automations/hh`. Вакансии с `testPresent: true` нельзя откликнуть POST'ом (`src/apply.py` их не берёт) — только браузером с прохождением теста.

**Разделение ролей — обязательно:**
- Главный тред: готовит пакет (шаги 1–2), запускает субагента, верифицирует и логирует (шаги 4–5).
- **Всю браузерную работу выполняет Agent-субагент с `model: opus`** (`subagent_type: general-purpose`, `run_in_background: false`). Не sonnet/haiku — тесты требуют суждений и честных развёрнутых ответов от лица кандидата.

## step-1: Вход

Список вакансий: из аргументов юзера или `testPresent: true` кандидаты последнего отчёта (`data/report-*.md`). Для каждой убедись, что полный текст скачан: `python3 src/fetch_vacancy.py --ids <id...>`. Проверь `data/applied.json` — уже откликнутые пропустить.

Затем главный тред ЧИТАЕТ полный текст каждой и отсеивает несовпадение стека (выстрадано@2026-07-13: сниппет-скоринг sonnet'ом пропускает жёсткие требования вроде «Angular от 6 лет», «Go — основной стек 6 лет» — честный тест там провалится; 3 из 22 выкинуты после чтения).

## step-2: Пакет

Собери `data/test-apply-package-<дата>.md`:
1. Секция фактов — НЕ дублируй, сошлись на `profile/candidate-facts.md` (субагент прочитает его сам).
2. По каждой вакансии: id, URL, выбор резюме (правило: senior-роли → «Senior fullstack engineer» EN; русскоязычные лид-роли → «Ведущий разработчик» RU; CTO/Head → «Head of IT, Principal developer» RU; hash'и в `src/apply.py::RESUMES`), сопроводительное письмо.
3. Письма пишет главный тред по правилам `profile/profile.md` (раздел «Сопроводительное письмо») с конкретными hook'ами из полного текста вакансии.

## step-3: Субагент (opus, headed)

Запусти Agent (model **opus**) с промптом-шаблоном ниже, подставив пути. Шаблон проверен@2026-07-09 — не упрощать, каждая инструкция выстрадана:

```
Задача: отправить отклики на вакансии hh.ru с обязательными тестами. Пользователь поручил пройти тесты от его имени. Python Playwright, HEADED (юзер видит окно).

Прочитай ПОЛНОСТЬЮ:
- /home/max/automations/hh/profile/candidate-facts.md — факты и правила ответов
- /home/max/automations/hh/data/test-apply-package-<дата>.md — вакансии, резюме, письма
- /home/max/automations/hh/data/details/<id>.json — полные тексты (контекст ответов)

Техника: python3 + playwright sync, launch(channel='chrome', headless=False), new_context(viewport 1440x900), куки из browser_cookie3.chrome(domain_name='hh.ru') через ctx.add_cookies (name/value/domain/path/expires/secure). Скрипты и скриншоты — в scratchpad-каталог сессии, не в репо. Пошагово: открыл → скриншот/дамп DOM → прочитал → следующий шаг. Таймауты 30с.

ВАЖНО (выстрадано):
- Скажи юзеру в начале НЕ кликать в открывающемся окне — параллельные ручные клики создают отклики вне спецификации (случилось@2026-07-09: ушёл отклик без письма и с не тем резюме).
- Если во время работы на вакансии появился отклик, созданный не тобой — НЕ отправляй дубль (hh: alreadyApplied), зафиксируй в отчёт, реши следующую вакансию.
- CAPTCHA → СТОП немедленно, скриншот, отчёт. Не решать.
- Элемент не находится с 2-3 попыток → скриншот, зафиксировать, дальше. Не долбить.

Процесс по каждой вакансии (последовательно, пауза 10с между):
1. Открой URL. 2. «Откликнуться» (может быть попап резюме / форма с тестом). 3. Выбери резюме из пакета. 4. Вставь письмо из пакета. 5. Тест: читай каждый вопрос; отвечай по фактам candidate-facts.md и тексту вакансии; типы: radio/checkbox — честный вариант; текст — 2-4 предложения; числа — из фактов. Вопрос без покрытия фактами и без честного «нет» → НЕ отправлять, вопрос дословно в отчёт, следующая вакансия. 6. Скриншот заполненной формы. 7. Отправь. Скриншот результата. 8. Верифицируй: hh.ru/applicant/negotiations?filter=all содержит vacancyId.

Только эти вакансии, никаких других действий. В конце browser.close().

Отчёт: по каждой — статус, вопросы теста ДОСЛОВНО + данные ответы, письмо/резюме, верификация, пути скриншотов.
```

## step-4: Верификация главным тредом

Не верь отчёту на слово: сверь сам через `verify_sent` (python: `sys.path.insert(0,'src'); from apply import verify_sent; from hh_session import make_session`) или GET negotiations. Совпадение vacancyId + желательно `hasResponseLetter=true`.

- Юзер кликнул в окне параллельно → см. ВАЖНО; письмо можно дослать сообщением в чат отклика (только по явному запросу юзера).
